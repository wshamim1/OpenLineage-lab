A complete setup for tracking data lineage using OpenLineage and Marquez with Podman.

## 🚀 Quick Start

### 1. Start Marquez Services

```bash
cd OpenLineage
./scripts/podman-up.sh --api-port 5050
```

**Note**: We use port 5050 instead of 5000 to avoid conflicts with macOS Control Center.

### 2. Verify Services

```bash
# Check services are running
podman ps

# Test the API
curl http://localhost:5050/api/v1/namespaces
```

### 3. Access Web UI

Open http://localhost:3000 in your browser.

## 📊 Services

| Service | URL | Description |
|---------|-----|-------------|
| **Marquez Web UI** | http://localhost:3000 | Visual lineage explorer |
| **Marquez API** | http://localhost:5050 | REST API for lineage data |
| **Marquez Admin** | http://localhost:5001 | Admin/health endpoints |
| **PostgreSQL** | localhost:5432 | Metadata storage |

## 🔧 Management Scripts

### Start Services
```bash
./scripts/podman-up.sh --api-port 5050
```

Options:
- `--api-port PORT` - Set API port (default: 5050)
- `--tag VERSION` - Use specific image version
- `--build` - Build from source

### Stop Services
```bash
./scripts/podman-down.sh
```

### Test Lineage
```bash
./scripts/test-lineage.sh
```

This creates sample lineage data you can view in the Web UI.

## 💡 Examples

### Python Integration

Run the Python example:
```bash
cd examples/python
pip3 install -r requirements.txt
python3 python-lineage-example.py
```

This demonstrates:
- Reading CSV data
- Processing/filtering data
- Writing output
- **Tracking complete lineage**

View results at http://localhost:3000 (search for `csv-processor`)

See [docs/PYTHON_LINEAGE_GUIDE.md](docs/PYTHON_LINEAGE_GUIDE.md) for detailed documentation.

## 🔍 Common Tasks

### View Logs
```bash
podman-compose logs -f
# or for specific service
podman logs -f marquez-api
```

### Restart Services
```bash
./scripts/podman-down.sh
./scripts/podman-up.sh --api-port 5050
```

### Check Service Health
```bash
curl http://localhost:5001/healthcheck
```

### Query API
```bash
# List namespaces
curl http://localhost:5050/api/v1/namespaces

# Get specific job
curl http://localhost:5050/api/v1/namespaces/NAMESPACE/jobs/JOB_NAME

# Get dataset
curl http://localhost:5050/api/v1/namespaces/NAMESPACE/datasets/DATASET_NAME
```

## ⚙️ Configuration

Configuration is stored in `.env` file:

```bash
# API Configuration
API_PORT=5050          # Changed from 5000 (macOS conflict)
ADMIN_PORT=5001
WEB_PORT=3000

# Database
POSTGRES_PORT=5432
POSTGRES_DB=marquez
POSTGRES_USER=marquez
POSTGRES_PASSWORD=marquez
```

## 🐛 Troubleshooting

### Port 5000 Already in Use
**Problem**: macOS Control Center uses port 5000

**Solution**: We use port 5050 instead (configured in `.env`)

### Services Not Starting
```bash
# Check Podman machine status (macOS)
podman machine list

# Start if needed
podman machine start

# Check logs
podman-compose logs
```

### Connection Refused
Make sure services are running:
```bash
podman ps
```

Should show: `marquez-db`, `marquez-api`, `marquez-web`
